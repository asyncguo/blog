    # Babel 如何解析 JSX

    Babel 解析 JSX 语法的插件：

    1. @babel/plugin-syntax-jsx

    能够让 Babel 解析 JSX 语法

    1. @babel/plugin-transform-react-jsx

    在 @babel/plugin-syntax-jsx 基础上，可以将 JSX 转换为 JS 能够识别的 createElement 格式

    react v17 版本之后使用全新的JSX转换 (opens new window)，即：新的 JSX 转换不会将 JSX 转换为 React.createElement，而是自动从 React 的 package 中引入新的入口函数并调用。

    全新的JSX转换对应于Babel v7.9.0提供的Automatic runtime (opens new window)，可通过修改@babel/preset-react的配置 runtime: classic | automatic

    1. classic: 不会自动导入（这也是react@17版本之前都需要在文件顶部手动导入import React from 'react'）
    2. automatic: 自动导入 JSX 转译函数 (自动导入require("react/jsx-runtime"))
    {
  "presets": [
    [
      "@babel/preset-react",
      {
        "pragma": "dom", // default pragma is React.createElement (only in classic runtime)
        "pragmaFrag": "DomFrag", // default is React.Fragment (only in classic runtime)
        "throwIfNamespace": false, // defaults to true
        "runtime": "classic" // defaults to classic
        // "importSource": "custom-jsx-library" // defaults to react (only in automatic runtime)
      }
    ]
  ]
}
